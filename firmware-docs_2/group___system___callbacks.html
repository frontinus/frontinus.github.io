Detailed Description

HAL interrupt and event callback implementations.

These functions are the glue between hardware interrupts and the RTOS, often by setting flags or putting data into queues.

void HAL_UART_RxCpltCallback ( UART_HandleTypeDef *  huart)

HAL UART Receive Complete Callback (Full DMA).

This callback is triggered when a DMA transfer is fully complete.

  • For UART4 (LoRa): Queues the received command for LoRaCommandTask and re-arms DMA.
  • For USART6 (Airbrakes): Copies new angle data and sets the RECEIVED_FLAG for Received_Pckg task. Re-arms DMA.
  • For USART1 (DaVinci): Copies new DaVinci packet and sets the RECEIVED_FLAG. Re-arms DMA.
    Parameters
    huartUART handle that finished receiving.

HAL UART Receive Event Callback (Idle Line).

This callback is triggered on UART Idle Line detection, suitable for variable-length messages like NMEA.

  • For UART5 (GNSS): Calls gnss_dma_handling to parse the new NMEA data.
  • If AVNBAY/NOSECONE, it also transmits gps_velocity to DaVinci (USART1).
    Parameters
    huartUART handle that detected the idle line.
    SizeNumber of bytes received since the last idle event.
